Note, in this template there are elements that rely on external images, and Jupyter Book specific markdown. This will be addressed when the assessment notebook (.pynb) file is added to the Jupyter Book. This is a combination of the assessment template and a set of instructions for the conversion.

Note: if the assessment does not include code (a literature review, paper or PQAR summary, etc), then this template can followed as a (.pynb) file, without any code cells and only embedded images.

Conversion to Jypyter Book format: the following steps need to be followed

  • Add the .ipynb or .md file to the data stream subfolder

  • Add logo line ![logo](../LogoLine_horizon_C3S.png)

  • Update title if needed

  • CHeck for missing sections or formatting, check methodology links etc

  • Download the embedded figure(s) from the rendered Github .ipynb or .md file, and add them to the data stream subfolder, adding the filename to the embedded image or figure, replacing ‘test_img.py’ (to be automated).

  • If a figure caption is provided it should be formatted in itallics below the image, or replace ‘Here is my figure caption’ in the figure markdown

  • Add the file to the table of contents .yml under the datastream, with the text to be displayed in the sidebar and datastream page links (the same as the title)

Unique quality assessment title (to be finalised by C3S), including [data stream]-[quality area]-for-[application area]#

We do not aim to be completely prescriptive about the title, it should be readable and descriptive, but also include some key tags to increase searchability, e.g. Seasonal forecasts bias assessment for impact models

  • Data stream: Satellite (observations), Insitu (observations), Reanalysis, Seasonal (Forecasts), Climate (projections)

  • Quality area: bias, forecast skill, completeness, intercomparison of X and Y, comparison to X, trend assessment, extremes, spatial/temporal resolution, reliability, accuracy, etc

  • Application area: impact models, risk assessment, climate monitoring, scientific study of X, X sector, ‘region’, ‘specific C3S application’, flood forecasting, etc

Production date: DD-MM-YYYY

Produced by: ‘institute’ and/or ‘author’, otherwise ‘EQC evaluator’

Quality assessment statement#

Key results and guidance including an answer to the user question - to be visually highlighted (in the Jupyter Book) - and with the title ‘Quality assessment statement’. This is the code:

```{admonition} These are the key outcomes of this assessment
:class: note
* Statement 1
* Statement 2
* Statement 3
* etc
```

This is the outcome, which will render properly in the Jupyter Book in blue box with an info icon:

These are the key outcomes of this assessment

  • Statement 1

  • Statement 2

  • Statement 3

  • etc

A ‘key figure’ could be included if appropriate (think of this as a ‘graphical abstract’, that may also be used as a thumbnail in the CDS or Jupyterbook). Since this image (or any other embedded image) is not generated directly from code, it needs to be saved externally and provided with the Notebook, or embedded.

There is an option for images to be dragged and dropped from an external file browser into the markdown cell (the image is then encoded with Base64, and no external file is needed - but this may not work universally or always render properly in a Jupyter Book). At the publication step this can be solved.

Basic markdown test ![Test image](test_img.png) (but no formatting options, so the image must be the right size):

Simple alternative with html <img src="test_img.png" width="300px"> (width controlled via width=”300px” as needed):

Please include a caption below the image in itallics. It should include attribution if it is taken from an external source (if the copyright allows reproduction!).

This MyST MD option allows a figure caption to be added and the image centered, but will not render properly in all Jupyter viewers but works when the Jupyter book is built!

```{figure} test_img.png
---
height: 150px
name: example-fig
---
Here is my figure caption!
```

Which will render like:

It will automatically be numbered, and can be referenced in the text like [text to link to the figure](example-fig), text to link to the figure. You can also make use of numbered referencing, like {numref} followed by the figure name in back ticks, giving Fig. 2, this can also be used for any other figures throughout the assessment which are not generated by code (but only works when the book is built). Any figures generated by code can have their own independent referencing. Again, the references will not render properly in all Notebook viewers/editors, but will work when the book html is generated.

Any references in the text should use ‘Fig. X’, which will then be compatible with the markdown figure numbering (but without active links).

Methodology#

A ‘free text’ introduction to the data analysis steps or a description of the literature synthesis, with a justification of the approach taken, and limitations mentioned.

Followed by a numbered list of the methodology and results, with the same headings as the sections under ‘Analysis and Results’. These should be links to the sections below using the format [Text to be displayed](#words-in-heading). Note that links will not work in the Jupyter Notebook editor, but you can test them in html output.

  • These headings can be specific to the quality assessment, and help guide the user through the ‘story’ of the assessment. This means we cannot pre-define the sections and headings here, as they will be different for each assessment.
